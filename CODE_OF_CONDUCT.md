# Code of Conduct

## Our Standard

We want InnerWarden to be a project where people can collaborate clearly, safely, and respectfully.

Examples of behavior that help:

- being respectful in disagreement
- assuming good intent while staying honest about risks
- giving actionable, technically grounded feedback
- welcoming contributors at different experience levels
- keeping security discussions responsible and calm

Examples of unacceptable behavior:

- harassment, intimidation, or personal attacks
- doxxing or publishing private information without permission
- insulting or demeaning language
- bad-faith disruption, trolling, or hostility
- encouraging unsafe or reckless use of the project without clear warnings

## Scope

This applies to repository discussions, issues, pull requests, and other project spaces.

## Enforcement

Maintainers may remove, edit, or reject comments, issues, commits, or contributions that violate this code of conduct.

## Reporting

If you experience or witness unacceptable behavior, contact the maintainer privately through the repository profile.
